# Лендинг «Тренеры Метеор Москва» — дизайн-документ

**Дата:** 2026-04-16
**Заказчик:** руководитель футбольного отделения школы Метеор Москва
**Источник данных:** Google Таблица «Анализ по тренеру / 2025-26»
(`https://docs.google.com/spreadsheets/d/1Ds11Kv8FV8vj2qBjXxfZDoIicuA0uB8ITMiOzWlWHfc/`)

## Цель

Внутренний визуальный дашборд для тренерского состава и руководства школы Метеор. Каждый тренер должен иметь возможность открыть страницу, выбрать своё ФИО и увидеть собственную годовую статистику. Все тренеры видят данные всех. Защита от посторонних — общий пароль.

## Аудитория

- **Тренеры школы (~21 человек)** — смотрят свои показатели и показатели коллег
- **Руководство** — общая картина по всем тренерам

## Ключевые принципы

- **Бесплатно** — никаких платных сервисов
- **Ручной ввод данных продолжается** — Google Таблица остаётся единственной точкой ввода
- **Автообновление** — изменили цифру в таблице → через минуту видно на сайте
- **Премиум-стиль** — лаконично, по-спортивному, не «детский» дизайн
- **Поддерживается легко** — добавили строку в таблицу → новый тренер появился сам

## Архитектура

Single-Page Application (SPA), один HTML-файл с встроенными CSS и JS.

```
┌─────────────────┐         ┌──────────────────┐         ┌─────────────────┐
│ Google Sheet    │  CSV    │  Static SPA      │  HTTPS  │     Coach       │
│ (dashboard_data │ ──────► │  (Netlify)       │ ──────► │     browser     │
│  tab, published)│         │  HTML/CSS/JS     │         │                 │
└─────────────────┘         └──────────────────┘         └─────────────────┘
```

**Стек:**
- Vanilla HTML + CSS + JavaScript (без фреймворков)
- Хостинг: Netlify (free tier)
- Источник данных: вкладка `dashboard_data` Google Таблицы, опубликованная как CSV
- Парсинг CSV — встроенный JS
- Графики — лёгкая библиотека (Chart.js, ~60kb)
- Шрифты — Google Fonts

**Почему vanilla, а не Astro/React:**
- Один файл проще передать и поддерживать
- Нет шага сборки, нет node_modules
- Размер итогового сайта < 200 KB
- Любые правки в будущем — открыл, поправил, залил

## Структура страниц

### 1. Страница входа (`/`)
- Чёрный фон во весь экран
- По центру логотип Метеор Москва (щит с буквой М)
- Поле «Пароль» + кнопка «Войти» (бордовая)
- При верном пароле — `sessionStorage.setItem("auth", "ok")` и переход на главную
- При неверном — короткая встряска поля + сообщение «Неверный пароль»
- Пароль хранится в коде в виде хэша (SHA-256), сравнивается на клиенте

### 2. Главная страница (`#/coaches`)
- Шапка:
  - Слева: логотип Метеор + надпись «Тренерский состав»
  - Справа: «Сезон 2025–26»
- Поле поиска по ФИО (мгновенная фильтрация по мере ввода)
- Сетка карточек тренеров:
  - Десктоп: 4 в ряд
  - Планшет: 3 в ряд
  - Телефон: 2 в ряд
- Карточка тренера:
  - Сверху — квадратная плашка-аватар с белыми инициалами (ФИО → инициалы фамилии и имени)
  - Цвет плашки — бордовый `#9F1239`
  - Под плашкой — ФИО (крупно, жирно)
  - Маленькая подпись «N детей» (главная цифра)
  - Hover: лёгкий подъём + бордовый бордюр
  - Клик — переход на страницу тренера

### 3. Страница тренера (`#/coaches/<slug>`)
- Шапка-баннер:
  - Чёрный фон с тонкой бордовой полосой
  - Слева — большая плашка инициалов
  - Рядом — крупное ФИО, под ним «Тренер | сезон 2025–26»
  - В правом верхнем углу — «← Все тренеры»
- Блок KPI-карточек (5–6 в ряд на десктопе, 2 в ряд на телефоне):
  - **Дети в группах** — `N`
  - **Лагерь** — `Факт / План` крупно, ниже `XX%` конверсии
  - **Мерч** — `N шт`
  - **Сборные команды** — `N`
  - **Кубок Метеора** — `Факт / План` + конверсия
  - **Лига Метеора** — `N команд`
- Каждая KPI-карточка с типом «План/Факт» — кликабельная, разворачивает разбивку по сезонам (Осень / Зима / Весна) под собой
- Под KPI — график динамики количества детей по месяцам (если данные накапливаются помесячно)

## Поток данных

### Источник
Существующая Google Таблица содержит «грязный» формат с объединёнными ячейками и двухуровневыми заголовками. Сайт не может её парсить напрямую.

### Решение: вкладка `dashboard_data`
Внутри той же Google Таблицы создаётся отдельная вкладка `dashboard_data` с плоской структурой. Каждая ячейка этой вкладки — формула, которая берёт значение из основной вкладки `2025-26`.

**Структура `dashboard_data`** (одна строка = один тренер):

| ФИО | kids | camp_plan_total | camp_fact_total | camp_plan_autumn | camp_fact_autumn | camp_plan_winter | camp_fact_winter | camp_plan_spring | camp_fact_spring | merch_total | teams | cup_plan | cup_fact | league_teams |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|

### Публикация
В Google Таблице: `Файл → Поделиться → Опубликовать в интернете → выбрать вкладку dashboard_data → формат CSV → Опубликовать`. Получается публичная (но скрытая, угадать невозможно) ссылка на CSV.

### Загрузка на сайте
1. При открытии главной страницы JS делает `fetch(CSV_URL)`
2. Парсит CSV в массив объектов `{fio, kids, camp_plan_total, ...}`
3. Кэширует в `sessionStorage` на 60 секунд (чтобы не дёргать таблицу при каждом клике)
4. Рендерит карточки

### Что вы делаете в обычной жизни
- Заполняете цифры в основной вкладке таблицы (как и сейчас)
- Формулы во вкладке `dashboard_data` пересчитываются автоматически
- Через минуту изменения видны на сайте

## Безопасность

### Защита паролем
- Один общий пароль на всю команду (тренеры + руководство)
- В коде сайта хранится только SHA-256 хэш пароля, не сам пароль
- Сравнение происходит на клиенте: введённый пароль хэшируется → сравнивается с хэшем в коде
- При совпадении — флаг авторизации в `sessionStorage`
- При неуспехе — сообщение, поле очищается

### Честное ограничение
Это «защита от случайных глаз», а не от целенаправленной атаки. Технически продвинутый человек может:
- Открыть инструменты разработчика и обойти проверку
- Найти CSV-ссылку в исходном коде и скачать данные напрямую

Для внутреннего инструмента школы это приемлемый компромисс. Если потом понадобится серьёзная защита — переделаем (например, через Netlify Identity или Cloudflare Access).

### Конфиденциальность данных
- Опубликованная CSV-ссылка — длинная случайная строка, угадать невозможно
- Сама Google Таблица остаётся приватной (опубликован только лист `dashboard_data`)
- На лист `dashboard_data` НЕ попадает: ФИО детей, телефоны родителей, суммы оплат — только агрегированные метрики тренеров
- HTTPS — включён автоматически у Netlify

## Визуальный стиль

### Палитра (из брендбука Метеор Москва)
- Бордовый основной: `#9F1239` (CMYK 8/100/75/39)
- Бордовый тёмный для акцентов: `#6B0F2A` (CMYK 11/100/69/51)
- Чёрный: `#000000`
- Белый: `#FFFFFF`
- Серый фон: `#F5F5F5`
- Серый текст-подпись: `#737373`

### Шрифты
- Заголовки и ФИО: **Bebas Neue** (или Druk) — спортивный гротеск
- Цифры в KPI: тот же Bebas, очень крупно
- Основной текст: **Inter**
- Все шрифты — Google Fonts (бесплатно, грузятся по CDN)

### Стилистика
- Тёмный «премиум-спорт»: много чёрного, акценты бордовым
- Карточки с тонкими рамками и лёгкой тенью
- Радиусы скругления: 4 px (строго)
- Hover-анимации сдержанные: подъём + бордовый бордюр
- Цифры — главные герои, всё остальное им служит

### Адаптивность
- Десктоп ≥ 1024 px: 4 карточки в ряд
- Планшет 768–1023 px: 3 в ряд
- Телефон < 768 px: 2 в ряд (KPI: 2 в ряд)

## Хостинг и деплой

### Платформа: Netlify
- Бесплатный план: безлимитный трафик для нашего размера, без рекламы, HTTPS включён
- Поддомен: `meteor-coaches.netlify.app` (или другое имя на выбор)
- Деплой через drag-and-drop (или GitHub-интеграция, если хочется версионирования)
- Свой домен в будущем подключается за 5 минут

### Резервный план
Если Netlify «не понравится» — тот же набор статических файлов работает на Vercel, GitHub Pages, Cloudflare Pages. Никакой жёсткой привязки.
